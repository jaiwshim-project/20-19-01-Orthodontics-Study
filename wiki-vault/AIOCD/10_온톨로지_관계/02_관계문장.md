# AIOCD 온톨로지 - 관계 문장 (25개)

온톨로지 관계 문장은 "주어 - 관계 - 목적어" 형식으로 정의됩니다.

---

## 그룹 1: AIOCD 내부 기능 흐름 (5개)

```
1. AI 부정교합분류(F1)는 세팔로메트리분석(F2)의 입력정보를 보강한다.
2. 세팔로메트리분석(F2)는 AI치료계획생성(F3)에 필요한 정량데이터를 제공한다.
3. AI치료계획생성(F3)는 3D치아이동시뮬레이션(F4)의 기초가 된다.
4. AI치료계획생성(F3)는 환자교육콘텐츠(F5)와 교정의임상가이드(F6)를 동시에 생성한다.
5. 케이스라이브러리(F7)는 교정의학습플랫폼(F8)의 학습자료로 활용된다.
```

---

## 그룹 2: AIOCD → GEO Score AI (4개)

```
6. AIOCD는 GEO Score AI의 진단 진입도구다.
   (GEO가 "진단 효율 낮음"을 지적 → AIOCD 도입 근거 제시)

7. AIOCD의 진단시간단축 성과(P1해결)는 GEO Score AI의 "효율성 개선" KPI로 측정된다.
   (AIOCD 도입 전후 비교)

8. GEO Score AI의 10대KPI진단결과는 AIOCD 도입의 ROI 근거를 제공한다.
   (병원장에게 투명한 성과 수치 제시)

9. AIOCD + GEO Score AI의 결합은 "병원 효율혁신 패키지" 상품으로 통합된다.
   (월 1200만 원 통합 패키지)
```

---

## 그룹 3: AIOCD → MedVo (6개)

```
10. AIOCD의 정확한진단과3D시뮬레이션은 외국인환자신뢰도를 높인다.
    (MedVo 도입 전 기초 신뢰 형성)

11. AIOCD가 생성한 치료계획데이터는 MedVo의 다국어상담설명 입력이 된다.
    (의료 정보 연계: AIOCD → MedVo)

12. AIOCD의 환자교육콘텐츠는 MedVo 상담 시 참고자료로 활용된다.
    (상담 효율성 증가)

13. MedVo의 외국인환자상담데이터는 AIOCD의 환자교육콘텐츠개선 주제를 제공한다.
    (실제 환자 질문 기반 콘텐츠 개선)

14. AIOCD + MedVo의 결합은 "글로벌환자확보패키지" 상품으로 통합된다.
    (월 1000만 원, 외국인 환자 3배 유입 기대)

15. AIOCD의 임상가이드(F6)는 MedVo 통역사의 의료용어 정확도를 높인다.
    (의료 정보 일관성)
```

## 그룹 4: AIOCD → Insight Bank Hub (4개)

```
16. AIOCD가 생성한 환자교육콘텐츠는 Insight Bank Hub의 기사화 소재가 된다.
    (자동 생성 콘텐츠 → 기사화)

17. Insight Bank Hub의 기사화된콘텐츠는 병원의 신뢰도를 강화하고, 결국 AIOCD의 환자신뢰도를 높인다.
    (선순환: 콘텐츠 → 기사 → 신뢰 → 환자 만족)

18. AIOCD + Insight Bank Hub의 결합은 "AI교정콘텐츠신뢰미디어패키지" 상품으로 통합된다.
    (교정학 콘텐츠의 SEO 최적화)

19. Insight Bank Hub의 SEO최적화기사는 병원의 온라인노출을 증가시켜, GEO Score AI의 마케팅KPI 개선에 기여한다.
    (3개 플랫폼 연쇄)
```

## 그룹 5: AIOCD → AX Ontology OS (3개)

```
20. AX Ontology OS는 병원의 업무프로세스를 분석하고, AIOCD 도입 시점과 방식을 최적화한다.
    (AX 진단 → AIOCD 커스터마이징)

21. AIOCD의 도입성과(환자만족도, 진료효율, 매출증가)는 AX Ontology OS의 "ROI개선" 지표로 활용된다.
    (성과 측정 및 다음 개선안 도출)

22. AX Ontology OS의 병원최적화제안은 AIOCD 사용 방식(기능선택, 워크플로우)을 병원 특성에 맞게 가이드한다.
    (개별 병원 맞춤화)
```

## 그룹 6: 크로스 플랫폼 관계 (3개)

```
23. AIOCD + MedVo + Insight Bank Hub의 통합은 
    "치과병원글로벌환자확보완전패키지"를 만든다.
    (진단 → 상담 → 신뢰 확산의 완전 사이클)

24. 4개 플랫폼(AIOCD + GEO + MedVo + AX)의 통합데이터는 
    "병원의 완전한 디지털혁신"을 가능하게 한다.
    (현황분석 → 프로세스개선 → 진단자동화 → 환자전환 → 성과측정)

25. AIOCD의 케이스라이브러리(F7)는 모든 플랫폼의 벤치마크 기준이 된다.
    (1000개 케이스 = 공통 데이터 자산)
```

## 관계 문장 작성 원칙

### 1. 명확성
- "강화한다" vs "관련이 있다" (명확한 동사 사용)
- 구체적인 데이터/결과 명시

### 2. 단방향 / 양방향 구분
- 단방향: "AIOCD → MedVo" (데이터 흐름)
- 양방향: "AIOCD ↔ GEO Score AI" (상호작용)

### 3. 비즈니스 임팩트
- 각 관계가 왜 중요한지 기대효과 명시

### 4. 선순환 구조
- 플랫폼 A의 결과 → 플랫폼 B의 개선 → 플랫폼 A의 성과 증가
