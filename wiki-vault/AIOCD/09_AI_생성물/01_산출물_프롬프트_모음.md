## 1️⃣ 통합 제안서 생성 프롬프트

### 대상: 병원장 (개인 교정 전문가)

```
AIOCD 위키의 다음 문서를 참고하여 
"서울 중요 교정 클리닉" 원장에게 제출할 AIOCD 제안서를 작성하라.

참조 문서:
- 01_플랫폼_한줄정의
- 02_해결하는_고객문제
- 05_핵심기능목록
- 12_영업제안논리 (병원장용 ROI 계산)
- 17_고객사례 (비슷한 규모 병원)
- 11_가격정책
- 25_플랫폼간_연결전략 (MedVo 연결 제안)

작성 조건:
1. 현재 병원의 문제점을 구체적으로 언급 (초진 20분 진단, 환자 설득 어려움)
2. AIOCD가 해결하는 방식을 3단계로 설명
3. 6개월·1년·3년 성과 예측을 수치로 제시
4. ROI 계산: (월 추가 매출 450만 원 - 월 구독 500만 원) = 월 순수익 추이
5. 다음 미팅 제안 ("금요일 30분 데모")

출력 형식: 
- 제목
- 1. 현황 분석 (3단락)
- 2. 해결책 (3단락)
- 3. 기대 효과 (표)
- 4. 가격 & ROI (표)
- 5. 다음 단계 (불릿)
```

## 2️⃣ 특허 초안 자동 생성 프롬프트

### 3개 핵심 특허 아이디어

```
AIOCD 위키의 다음 문서를 기반으로 특허 출원 가능성이 높은 3개 기술을 분석하고, 
각각의 특허 명세서 초안을 작성하라.

참조 문서:
- 07_기술구조
- 08_데이터구조
- 14_특허가능요소
- 02_플랫폼특화모듈/* (모든 특화 모듈)
- 25_플랫폼간_연결전략 (다른 플랫폼과의 결합 특허)

작성 대상 특허 3개:
1. "AI 기반 교정진단 자동화 시스템"
   (부정교합 자동분류 + 세팔로 정량화 + 치료계획 생성)
2. "치아 이동 3D 시뮬레이션 및 임상가이드 자동 생성 방법"
   (시뮬레이션 엔진 + 단계별 가이드 연계)
3. "AI 진단 기반 외국인 환자 상담 자동화 시스템"
   (AIOCD + MedVo 통합 특허)

각 특허마다 작성:
- 발명의 명칭
- 기술 분야
- 배경 기술 (문제 정의)
- 기존 기술의 한계
- 발명의 목적
- 핵심 구성 (5~7개 모듈)
- 작동 흐름 (입력→처리→출력)
- 기술적 효과 (3~5개)
- 구체적 실시예
- 청구항 후보 (5~10개 청구항)
```

## 3️⃣ IR (투자 피치) 자료 생성 프롬프트

```
AIOCD 위키를 기반으로 시리즈 A 투자자에게 제출할 IR 자료 구조를 작성하라.

참조 문서:
- 01_플랫폼_한줄정의 (투자자용 설명)
- 02_해결하는_고객문제
- 03_핵심고객군
- 06_차별화포인트
- 10_수익모델
- 15_향후확장로드맵
- 16_경쟁사비교
- 20_KPI_성과지표
- 25_플랫폼간_연결전략 (투자 포인트)

출력 형식 (12 슬라이드):
1. 표지 (AIOCD 한 줄 정의)
2. 문제 정의 (교정학의 3가지 근본 문제)
3. 시장 기회 (교정시장 규모, 성장률, 미충족 수요)
4. 솔루션 (AIOCD 5가지 핵심 기능)
5. 제품 데모 구조 (사용자 여정)
6. 기술 차별성 (경쟁사 대비 우위 5가지)
7. 비즈니스 모델 (3가지 가격 정책, 월/연 매출)
8. 경쟁 우위 (WebCeph, Invisalign vs AIOCD)
9. 고객 사례 (도입 병원의 성과)
10. 성장 로드맵 (6개월/1년/3년 마일스톤)
11. 투자 요청 (자금 규모, 사용처)
12. 연락처 & QA

핵심 수치:
- 시장: 한국 1만5천 교정의 × 월 500만 원 = 월 75억 원 매출
- ROI: 10개월 내 투자 회수
- 성장: 연 100% 성장 시 3년 후 누적 180억 원 매출
```

## 4️⃣ 블로그·기사 자동 생성 프롬프트

### A. 교정의 대상 블로그 글

```
AIOCD 위키를 기반으로 교정의가 읽을 블로그 글 5개를 작성하라.

각 글의 주제:
1. "2026년 교정진단, 왜 AI가 필수인가" (교육)
2. "초진 진단 20분 → 3분으로 단축하는 방법" (시간 관리)
3. "AI 시뮬레이션으로 환자 만족도 80% 달성하기" (영업)
4. "신입 교정의 교육, 6개월로 끝낼 수 있다" (HR)
5. "외국인 환자 3배 유입하는 병원의 비결" (글로벌)

각 글 작성:
- 제목 (SEO 최적화)
- 리드문 (30초 요약)
- 목차 (3~5개 섹션)
- 본문 (1,500단어)
- 결론 (행동 유도)
- 참고 문서 (AIOCD 위키 링크)

조건:
- AI 인용 최적화 (E-E-A-T 구조)
- 교정의의 실제 문제와 해결책 중심
- 수치로 설득 (효과 정량화)
```

### B. 환자 대상 블로그 글

```

주제:
1. "나의 치아는 앞으로 어떻게 움직일까? (3D 시뮬레이션 가이드)"
2. "교정 진단, 정말 정확한가? (AI 진단의 신뢰성)"
3. "교정 기간과 비용을 미리 알 수 있다면? (투명한 진료)"

각 글 작성:
- 일반인도 이해하는 설명
- 3D 이미지/다이어그램 필수
- 실제 환자 체험담 포함
- 이미지 중심 (텍스트 최소화)
```

## 📋 프롬프트 사용 방법

### Step 1: 위키 준비
- 해당 참조 문서가 최신 내용인지 확인
- 필요하면 위키 문서 업데이트

### Step 2: 프롬프트 선택
- 필요한 산출물에 맞는 프롬프트 선택
- 프롬프트의 "참조 문서" 확인

### Step 3: 프롬프트 실행
```bash
# Claude 또는 GPT에 다음 형식으로 입력:
[프롬프트 전체 복사]
[위키 내용 또는 링크 추가]
[추가 지시사항 있으면 입력]

# 예시:
"위 프롬프트를 사용해 서울 강남 교정 클리닉용 AIOCD 제안서를 만들어줘.
병원은 현재 월 100명 초진, 진단시간 22분, 환자 전환율 65%, 
월 진료료 7500만원을 올리고 싶어."
```
