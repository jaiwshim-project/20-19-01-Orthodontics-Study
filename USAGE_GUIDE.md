## 3가지 사용 시나리오

### 🎯 시나리오 1: 플랫폼 중심 학습 (가장 간단)

**상황**: 빠르게 학습을 진행하고 싶은 경우

**플로우**:
```
1. 플랫폼 홈 → 📚 자료 → 슬라이드/교과서 학습
2. 💡 개념정리 → 개념 저장
3. ⏱️ 진도 페이지 → 진도 입력 + 간단한 노트 작성
4. (선택) 📥 JSON 다운로드 → 백업
```

**장점**: 모든 것이 한 곳에서 해결
**단점**: 깊이 있는 정리 어려움

---

### 🎯 시나리오 2: Obsidian 중심 학습 (가장 깊이 있음)

**상황**: 개념을 깊이 있게 정리하고 싶은 경우

**플로우**:
```
1. Obsidian Vault 열기
2. Phase 1 폴더 → 01_EZ_개념.md 참고
3. 학습노트_Template.md 복사 → 학습 내용 정리
4. 여러 개념을 연결하며 자신의 이해 기록
5. (선택) 플랫폼의 진도만 주간에 한 번씩 업데이트
```

**장점**: 깊이 있고 체계적인 학습
**단점**: 진도 관리가 분산됨

---

### 🎯 시나리오 3: 하이브리드 학습 (추천 ⭐)

**상황**: 효율성과 깊이를 모두 원하는 경우

**플로우**:
```
학습 데이 (예: 월요일)
├─ 09:00 ~ 10:00: 플랫폼 → 📚 자료 접근 (슬라이드/교과서)
├─ 10:00 ~ 11:30: Obsidian 열기 → 학습노트.md에 정리
├─ 11:30 ~ 12:00: 플랫폼 → 💡 개념정리에 핵심 저장
└─ 12:00 ~ 12:30: 플랫폼 → ⏱️ 진도 입력 (1.5시간 기록)

주간 정리 (예: 금요일)
├─ Obsidian → 케이스분석.md로 사례 분석
├─ 플랫폼 → 💭 아이디어에 개선안 기록
└─ 플랫폼 → 📥 JSON export (진도 백업)
```

**장점**: 빠른 진도 + 깊은 이해
**단점**: 조금 복잡할 수 있음

## 단계별 상세 가이드

### 📌 Step 1: 플랫폼 첫 방문

#### 1-1) 통합 허브에서 시작
```
🏠 wiki-vault-index.html 열기
    ↓
우상단의 "🌐 웹 플랫폼" 섹션 확인
    ↓
"🏠 홈 - 플랫폼 개요" 링크 클릭
```

#### 1-2) 플랫폼 홈에서 확인할 것
- ✅ 90개 학습 자료 소개
- ✅ 9개 페이지 및 각 기능
- ✅ Phase 1-3 개요
- ✅ 기대 효과

#### 1-3) 학습경로 확인
```
🎓 학습경로 페이지
├─ Phase 1 학습 목표 + 추천 자료 표
├─ Phase 2 학습 목표 + 추천 자료 표
└─ Phase 3 학습 목표 + 추천 자료 표
```

---

### 📌 Step 2: 자료 접근하기

#### 2-1) 📚 자료 페이지 접근
```
네비게이션 → 📚 자료 클릭
    ↓
2개 탭 표시:
├─ 김용을 원장 자료 (67개)
│  ├─ 슬라이드 45개
│  ├─ 교과서 12개
│  ├─ 발표자료 7개
│  └─ PDF 3개
│
└─ 기타 자료 (23개)
```

#### 2-2) 원하는 자료 검색
```
🔍 검색창에 입력 (예: "EZ", "성장", "공간분석")
    ↓
필터링된 결과 확인
    ↓
자료 클릭 → 미리보기
```

#### 2-3) 자료 확인
```
슬라이드 → 이미지로 표시
교과서 → 다운로드 링크
이미지 → 확대해서 보기
```

---

### 📌 Step 3: 개념 정리하기

#### 3-1) 💡 개념정리 페이지
```
4개 탭 활용:
├─ 📚 기본 개념 (평형존, 5방향 분석)
├─ 🔍 진단 (공간분석, 부정교합 분류)
├─ 🏥 치료계획 (발치 판단, 치료 순서)
└─ 📖 기타 (해부학, 습관 영향)
```

#### 3-2) 개념 저장하기
```
예시: "평형존" 개념 저장

1) 각 탭에서 내용 읽기
2) 하단의 "💡 개념 저장" 입력란 클릭
3) 자신의 이해를 문장으로 작성
   예: "평형존은 정상 치열을 유지하는 안정적 영역이다"
4) [저장] 버튼 클릭
5) localStorage에 자동 저장 (브라우저 종료해도 유지)
```

---

### 📌 Step 4: 진도 추적하기

#### 4-1) ⏱️ 진도 페이지 열기
```
네비게이션 → ⏱️ 진도 클릭
    ↓
화면 구성:
├─ ⏱️ 학습 시간 관리
├─ 📊 Phase별 진도
├─ 🎓 전체 학습 진도
├─ 📈 학습 통계
├─ 📝 학습 기록
└─ 🔄 데이터 관리
```

#### 4-2) 학습 시간 입력
```
예시: 오늘 2시간 학습함

"이번 세션 학습 시간" 입력란
    ↓
2.0 입력 (또는 0.5, 1.5 등)
    ↓
[➕ 추가] 버튼 클릭
    ↓
자동으로 "총 학습 시간"에 누적됨
```

#### 4-3) Phase 진도율 입력
```
예시: Phase 1을 50% 완료함

Phase 1 입력란
    ↓
50 입력
    ↓
[업데이트] 버튼 클릭
    ↓
프로그레스 바 자동 업데이트
├─ Phase 1 프로그레스 바: 50%
├─ 전체 진도율: 자동 계산
└─ 진도 데이터 localStorage 저장
```

    ↓
자동으로 .obsidian-wiki 폴더 열림

**방법 B: wiki-vault-index.html에서**
```
통합 허브 → "🧠 Obsidian Vault" 섹션
    ↓
"Vault 열기 (파일 탐색기)" 링크 클릭
    ↓
자동으로 폴더 열림
```

```

│  ├─ 06_발치판단.md

3) 이름: "2026-05-22_EZ개념_학습노트.md"로 변경
4) 템플릿 내용 수정:

   updated: 2026-05-22
   tags: [phase1, EZ, 개념]
   ---

   ## 📖 핵심 내용
   
   평형존(Equilibrium Zone)은...
   [자신의 이해를 자유롭게 작성]
   
5) 저장 (Ctrl+S)
6) 이 노트는 .obsidian-wiki에 자동 저장됨
```

## 자주 하는 질문

### Q1: 플랫폼과 Obsidian 중 뭘 먼저 써야 하나요?

**A**: 플랫폼부터 시작하세요!
- ✅ 플랫폼에서 자료에 접근하고 진도를 입력
- ✅ 그 다음 Obsidian에서 상세 노트 작성
- ✅ 이렇게 하면 전체 맥락이 명확함

### Q2: 학습 노트를 플랫폼에만 저장해도 되나요?

**A**: 네, 가능합니다!
- 플랫폼의 "📝 학습 기록" 섹션에서 간단한 노트 작성 가능
- 하지만 **깊이 있는 학습**을 원하면 Obsidian 추천
- 짧은 메모 → 플랫폼
- 상세 정리 → Obsidian

### Q3: 진도 데이터를 잃어버리면?

**A**: 정기적으로 내보내기하세요!
- ⏱️ 진도 페이지 → 매주 금요일 JSON 다운로드
- 안전한 폴더에 저장 (구글 드라이브, 클라우드 등)
- localStorage는 브라우저 데이터 삭제 시 초기화됨

### Q4: Obsidian 앱이 없으면?

**A**: 파일 탐색기로도 가능합니다!
- 마크다운 파일은 메모장, VS Code 등에서도 열 수 있음
- Obsidian 설치하면 더 좋음 (연결, 백링크 등)
- 설치 안 해도 기본 기능은 다 사용 가능

### Q5: 여러 디바이스에서는?

**A**: 현재는 로컬 저장이므로:
- 같은 컴퓨터: localStorage + 파일 공유로 해결
- 다른 컴퓨터: JSON export → 이메일/클라우드로 전달
- 향후: 클라우드 동기화 기능 추가 가능

### Q6: Phase를 순서대로 안 하면?

**A**: 가능하지만 권장하지 않습니다!
- Phase 1: EZ 기본 개념 (필수)
- Phase 2: Phase 1 이후 진행 추천
- Phase 3: Phase 2 완료 후 진행
- 순서: 개념 → 응용 → 개발

### Q7: 케이스분석은 언제부터?

**A**: Phase 2부터 시작하세요!
```
Phase 1: 개념만 이해
Phase 2: 케이스분석.md 템플릿으로 케이스 분석 시작
Phase 3: 플랫폼 코드와 케이스 연결
```
