## 시작하기

### 1. 플랫폼 열기
```
파일 위치: C:\01 클로드코드\000 특허-저작권\01 치과 교정학\learning-platform.html

방법:
- 파일 탐색기에서 learning-platform.html 더블클릭
  → 기본 브라우저로 자동 열림
- 또는 Chrome/Edge/Firefox 등에서 파일을 드래그 앤 드롭
```

### 2. 브라우저 환경
```
권장: Chrome 90+, Edge 90+, Firefox 88+

주의:
- 로컬 파일이므로 localStorage 지원 필수
- JavaScript 활성화 필수
- 팝업 차단 기능 비활성화 권장
```

### 3. 첫 실행
```
첫 실행 시 자동으로:
- localStorage에 데이터 저장소 생성
- Phase 1 활성화 (Phase 2, 3은 locked)
- 홈 탭 표시

모든 학습 진도와 개념은 자동 저장됨!
```

## 플랫폼 개요

### 목표
```
교정학 74개 학습 자료 → Phase별 단계적 학습 
→ 플랫폼 독립적 개발 능력 확보
```

### 학습 구조
```
┌─────────────────────────────────────────────────────┐
│ 📚 자료 (74개: 이미지, PPTX, 영상, 문서)              │
├─────────────────────────────────────────────────────┤
│ Phase 1: 기초 이해 (1-2주)                           │
│   교정학 ABC 습득 + 진단 도구 5가지                   │
├─────────────────────────────────────────────────────┤
│ Phase 2: 고급 이해 (2-3주)                           │
│   4대 AI 엔진 임상 기초 + 플랫폼 API 분석              │
├─────────────────────────────────────────────────────┤
│ Phase 3: 플랫폼 적용 (2-3주)                         │
│   버그 수정 + 알고리즘 개선 + 신규 기능 기획           │
└─────────────────────────────────────────────────────┘
```

### 4대 AI 엔진
```
① Extraction AI: 발치 판단 (공간분석)
② Growth Prediction: 성장 예측 (CVMS, 세파로)
③ Facial Simulation: 안모 시뮬레이션 (미적 분석)
④ Recurrence Prediction: 재발 예측 (위험 인자 분석)

각 엔진마다 Phase 2에서 깊이 있게 학습 및 분석
```

### 📝 탭 4: 개념정리 (Concepts)

**기능**: 학습한 개념을 정리하고 저장

**미리 제공되는 예시 개념** (10개):
```
1. Angle 분류 - 상악 제1대구치 관계로 부정교합 분류
2. 정상 교합 - 상악 제1대구치가 정상 위치
3. Class I - 제1대구치 관계는 정상, 치아만 문제
4. Class II - 하악골이 상악골에 비해 뒤로 위치
5. Class III - 하악골이 상악골에 비해 앞으로 위치
6. Deep Bite - 수직 중첩 > 4mm
7. Open Bite - 상 전치와 하 전치 사이 공간
8. CVMS - 경추 성숙도로 성장 시기 판정
9. SNA 각도 - 상악골 위치 (정상: 80-82°)
10. SNB 각도 - 하악골 위치 (정상: 78-80°)
```

**새 개념 추가하기**:
```
1. "+ 새 개념 추가" 버튼 클릭
2. 폼 작성:
   ├─ 개념명 (예: "ANB 각도")
   ├─ 정의 (한 문장)
   ├─ Phase (1/2/3)
3. 저장 버튼
4. 개념 목록에 자동 추가 & localStorage 저장
```

**활용 방법**:
```
Phase 1 학습 중:
- 매일 배운 개념 1-2개씩 추가
- 목표: Phase 1 완료까지 100개 개념

Phase 2 학습 중:
- 4대 AI 엔진별 개념 50개 추가

Phase 3 학습 중:
- 플랫폼 개선 아이디어 관련 개념 추가
```

## 자주 묻는 질문

1. learning-platform.html 열기

**A**:
- Phase 2: Phase 1 완료 버튼 클릭 후 활성화
- Phase 3: Phase 2 완료 버튼 클릭 후 활성화

Phase 1: 10-15시간 (1-2주)

─────────────────────

### Q7. 브라우저를 바꾸면 데이터가 사라지나요?

**A**:
네, 현재는 각 브라우저의 localStorage에만 저장됩니다.

해결 방법:

## 📞 팁 & 트릭

### 학습 효율을 높이는 방법

**1. 다중 매체 활용**
```
① 영상 보기 (이해)
② 슬라이드 읽기 (정보)
③ 다이어그램 그리기 (확인)
④ 개념 정리 (기억)
```

**2. 3회 반복 원칙**
```
처음: 전체 개요 이해 (배속 1.5배로 빠르게)
두번째: 세부 내용 학습 (메모하며)
세번째: 핵심만 복습 (15분)
```

**3. 손으로 쓰기**
```
다이어그램을 종이에 손으로 그리기
수치 암기할 때 손가락으로 트레이싱
이러한 활동이 뇌 기억 네트워크 형성에 효과적!
```

**4. 임상 연결**
```
Phase 1-2를 배우며 생각:
"실제 환자에게 이것을 어떻게 설명할까?"
"이 진단을 어떻게 치료 계획으로 이어갈까?"
```

### 막혔을 때

```
문제 1: Phase 1 개념이 이해 안 됨
→ 해결: PHASE1_기초이해.md의 "개념 정리" 섹션 다시 읽기

문제 2: 자료가 어디 있는지 모름
→ 해결: 자료(📚) 탭 검색창에서 파일명 검색

문제 3: Phase 2 시작하고 싶은데 Phase 1이 안 끝남
→ 해결: Phase 1 완료 버튼을 일단 눌러 언락
        나중에 다시 돌아와 완성해도 됨

문제 4: 개념을 너무 많이 만들었어
→ 해결: localStorage에서 수정 불가능하지만
        장기 메모리에 도움이 됨! 그냥 진행 괜찮음
```

### 고급 팁

```
1. 개념 정리 시 "관련 자료" 적기
   → Phase 3에서 아이디어 연결할 때 유용

2. 아이디어 작성 시 구체적으로
   → 나중에 코드 수정할 때 참고 가능

3. 진도 탭 자주 확인
   → 시각적 피드백으로 동기 유지

4. Phase 3에서는 메모장 준비
   → 코드 분석 시 필기할 공간 필요
```

## 📞 문제 해결

### 데이터가 사라졌어요

```
증상: 브라우저를 닫았다 다시 열었는데 개념이 없음

원인: 
1. 다른 브라우저 사용
2. 시크릿/프라이빗 모드였음
3. 브라우저 캐시 삭제함

해결:
1. 같은 브라우저 사용
2. 일반 모드 사용
3. 정기적 백업 (스크린샷)
```

### 플랫폼이 느려요

```
원인: 브라우저 메모리 부족

해결:
1. 다른 탭 모두 닫기
2. 브라우저 재시작
3. 크롬 캐시 삭제
4. 다른 브라우저 사용해보기
```

### 버튼이 안 눌려요

```
원인: JavaScript 미활성화 또는 플러그인 간섭

해결:
1. JavaScript 활성화 확인
2. 확장 프로그램 비활성화
3. 다른 브라우저로 시도
```
